# 主頁懸浮視窗入口畫面核對紀錄

本紀錄核對主頁改為懸浮視窗入口後的靜態版面。核對使用專案開發預覽，並以全頁截圖檢視桌面與手機尺寸。

| 版面 | 視窗尺寸 | 主頁可見的主要操作 | 核對結論 |
| --- | ---: | --- | --- |
| 桌面版 | 1280 × 720 | 「開啟懸浮視窗」 | 標題、說明文字與唯一主要按鈕置中且清楚可見。未顯示主頁麥克風錄音按鈕、錄音狀態、辨識文字或轉換文字。 |
| 手機版 | 375 × 812 | 「開啟懸浮視窗」 | 唯一主要按鈕完整位於可視範圍，文字沒有裁切；未顯示主頁麥克風錄音按鈕、錄音狀態、辨識文字或轉換文字。 |

| 已確認移除的主頁元素 | 結果 |
| --- | --- |
| 「按住麥克風開始說話」提示 | 未顯示 |
| 錄音麥克風按鈕與處理狀態 | 未顯示 |
| 辨識文字與轉換文字結果區 | 未顯示 |

> 本次為靜態版面核對。實際麥克風權限、按住錄音、放開處理及 Document Picture-in-Picture 開啟流程，仍須在支援該功能並具備麥克風的 Chrome 裝置上進行實機驗證。
