# 互動按鈕方向標記移除核對紀錄

本次更新手動移除了主頁「開啟懸浮視窗」淺藍互動按鈕右側的方向標記，保留左側音量圖示、漸變光暈、點擊功能與鍵盤焦點樣式。

| 核對項目 | 結果 |
| --- | --- |
| 桌面版 | 1280 × 720；按鈕不再顯示右側方向標記，左側圖示及文字清楚置中，光暈層次完整。 |
| 手機版 | 375 × 812；按鈕文字和左側音量圖示完整可見，沒有裁切、溢出或多餘方向標記。 |
| 自動化檢查 | 7 個測試檔、39 項測試均通過；主頁回歸測試明確驗證不渲染 `hero-glow-button__arrow` 或 `↗`。 |
| 型別檢查 | TypeScript 一次性檢查通過。 |

> 已移除僅服務於方向標記的 CSS 規則，不影響按鈕光暈、懸停、按壓、焦點或減少動態偏好行為。

## 圖示與文字間距核對

移除方向標記後，按鈕內容容器已明確設定 `justify-content: center` 及 `gap: 0.875rem`。重新截取的桌面與手機畫面均顯示左側音量圖示與文字作為單一置中群組，兩者之間保留穩定、清楚的留白；按鈕右側不再有為方向標記預留的空間。

| 版面 | 內容群組結果 |
| --- | --- |
| 1280 × 720 | 圖示與「開啟懸浮視窗」置中於按鈕中段，右側沒有空白標記區。 |
| 375 × 812 | 圖示與文字完整置中、保持一致留白，沒有裁切或水平溢出。 |
